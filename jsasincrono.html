Assíncronicidade

Vamos trabalhar com Promises, como manipular e usar async e await

Assíncrono no JS permite requisitar uma informação a um servidor enquanto seu código continua rodando, não ficando travado ate o servidor enviar a resposta para depois fazer algo

Promises, é um objeto de processamento assíncrono, seu valor é desconhecido ela pode ser resolvida ou rejeitada, ex. você vai comprar um ingresso do cinema para tal dia, você tem a promessa de ver um filme mas caso algum imprevisto aconteça você não ira ver o filme você não vera o resultado.

Uma promise pode ter 3 estados, 1 pending (pendente), 2 fulfilled (completado), 3 rejected (rejeitado)

Async e await

Essas funcões existem para definir funções assíncronas, quando vamos resolver promises eu preciso dizer que essa função é assíncrona para que eu possa utilizar a palavra await que vai parar meu código para resolver a promisse só depois que tiver resolvido meu código vai continuar rodado.

Lembre-se que uma promise vai retornar uma outra promise
